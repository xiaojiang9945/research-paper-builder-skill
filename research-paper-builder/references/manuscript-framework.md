# Manuscript Framework

Use this reference when planning or restructuring a research article.

## Journal And Reference-Paper Alignment

Before major writing, extract these features from target instructions or reference papers:
- article type and section order,
- approximate word count and paragraph density,
- number and style of display items,
- figure-caption structure,
- Methods placement and detail level,
- reference count and citation format,
- how strongly the journal expects concise claims versus broad interpretation.

Do not copy reference-paper wording. Match structural conventions and scientific pacing.

## Manuscript Passport

For a full manuscript package, maintain a compact state file in the output folder. It is a working artifact, not part of the formal manuscript.

Recommended fields:
- route mode, article type, target journal and target style;
- one-sentence argument contract;
- source-material inventory with file names, dates and access status;
- literature status: search log path, reading matrix path, number fully read, number screened and unresolved gaps;
- data inventory: raw data, processed data, analysis scripts, source data and excluded files;
- figure registry: figure number, panel list, result question, source data, statistic/model and caption claim;
- claim registry status: numerical, factual, comparative, trend and causal claims checked, unchecked and blocked;
- citation status: reference style, bibliography source, locator availability and unresolved metadata;
- author actions: missing accessions, funding, author details, permissions, repository deposits and journal-specific items;
- validation log: scripts run, QC pass count, visual inspection status and package path.

Use the passport as the resume boundary for long work. Update it after literature intake, figure redesign, major rewriting, citation checks and final packaging. Keep process labels, QC scores and author actions out of manuscript text, captions and figures.

## Default Research Article Structure

Title:
- Prefer 10 to 18 words.
- State the system, main phenomenon and conceptual advance.
- Avoid inflated claims such as "novel mechanism" unless directly demonstrated.

Abstract:
- Usually 220 to 280 words for a full research article unless the journal specifies otherwise; shorter abstracts are acceptable only for strict journal limits or section-specific drafting.
- Cover problem, approach, key evidence, main conclusion and significance.
- Include concrete details such as sample size, major methods, strongest result types and restrained interpretation.
- Avoid unsupported mechanism language and generic importance statements.

Keywords:
- Use 5 to 7 terms unless the journal specifies another limit.
- Include organism or system, method, process, and application domain.

Introduction:
- Move from field-level problem to knowledge gap to study design.
- A strong sequence is: importance, current understanding, unresolved gap, why available data can answer it, and what this study tests.
- Cite the background claims that establish importance, mechanism and prior method precedent.
- Avoid literature dumping. Each paragraph should narrow the problem.
- For a full manuscript, aim for roughly 1,100 to 1,300 words unless the journal requires compression. About 1,200 words is the default for a developed research-article background.
- Every paragraph should have citation support, and every citation must appear in the reference list.

Results:
- Use 4 to 8 subsections for a full article when data volume supports it.
- Each subsection should support one result theme and connect to one main figure or a clear figure group.
- Use integrated, multi-panel figures as the Results backbone. A full article should normally include figure callouts, panel-level interpretation and source-data links, not text-only Results.
- Start with what was measured or compared, report the result, then interpret cautiously.
- End each subsection with a concise conclusion that does not exceed the evidence.

Discussion:
- Start with the main model or finding, not a full recap.
- Compare the result with relevant literature.
- Distinguish mechanism, association, hypothesis and future testing.
- Include limitations that affect interpretation, not generic disclaimers.
- End with the practical or conceptual implication.

Methods:
- Make the work reproducible.
- Include samples, experimental design, data preprocessing, statistics, visualization, software versions and accession information where available.
- Put exploratory and confirmatory analyses in separate language when relevant.

Back Matter:
- Data availability should identify raw data, processed data, code and accession status.
- Acknowledgements should include funding and technical help.
- Author contributions should map people to work categories.
- Competing interests should be explicit.
- References must be cited in the text and formatted consistently.

## Section Balance

For a journal-style article with full Methods, a useful starting balance is:
- Abstract: 3 to 5 percent of text.
- Introduction: 12 to 18 percent.
- Results: 35 to 45 percent.
- Discussion: 18 to 25 percent.
- Methods: as detailed as needed, often 25 to 40 percent when included in the main file.

Adjust to the journal. Nature-style formats often demand high compression in main text and move technical detail to Methods or supplementary material.
